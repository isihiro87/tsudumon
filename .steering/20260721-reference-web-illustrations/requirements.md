# 要求内容

## 概要

つづもん「参考書 Web版」の各セクション（ステップ）に、内容に関連した挿絵を1枚ずつ横帯で表示し、歴史ぎらいの中学生でも「深く・楽しく」読めるようにする。

## 背景

- 現状の参考書Web版は、各トピック冒頭に1枚のヒーロー画像があるだけで、本文セクション（ステップ）には画像がほぼない。
- 文字中心のため、視覚的な手がかりが乏しく、記憶の定着・読む楽しさに欠ける。
- 「1ページ（＝Web版の1ステップ）に1枚程度」の関連画像があると、理解と没入が深まる。

## 実装対象の機能

### 1. セクション横帯挿絵（Web版）
- `reference/{章}.json` の各 section に `image` / `imageCaption` フィールドを追加できるようにする。
- `generate_reference_web.py` が、section 画像を本文の上（ステップ内）に横帯（フル幅）で表示する。
- 画像が無いセクションは従来どおり画像なしで表示（後方互換）。

### 2. 画像素材の用意（歴史④のみ・パイロット）
- 著作権的に問題ない実物写真（PD/CC0/CC BY）は積極活用。既存 assets も再利用。
- 概念図・シーン・肖像など、フリー素材で適さないものは codex（image-2）でオリジナル生成。
- 画像内に日本語が必要な図（関係図など）は image-2 の日本語生成を活用。

## 受け入れ条件

### セクション横帯挿絵（Web版）
- [ ] section に `image` を書くと、そのステップに横帯画像＋任意キャプションが出る
- [ ] `image` 無しのセクションはエラーにならず画像なしで表示される
- [ ] PDF生成側（generate_reference_book.py）は section.image を無視しても壊れない
- [ ] `--deploy 04` で歴史④のみ再生成でき、画像が img/ にコピーされる

### 画像素材（歴史④）
- [ ] 歴史④の全セクションに関連画像が1枚ずつ割り当てられている
- [ ] 実物写真は出典・ライセンスを credits.json に記録
- [ ] codex生成画像はトーンが揃い、文字化けした日本語が無い

## 成功指標

- 歴史④の各ステップに関連画像が表示され、見た目のトーンが破綻しない
- ユーザーがブラウザで歴史④を1単元通して読み、画像が内容と合致していると確認できる

## スコープ外

- PDF版参考書（generate_reference_book.py）への画像追加 … ユーザー方針で当面PDF版は使わないため
- 歴史①②③⑤〜⑲ … パイロット確認後に横展開
- 問題集Web版への画像追加
- 音声・アニメーション等の追加
