# 記述問題 追加タスク（共通指示書）

対象の章データJSON（`books/{章}.json`）に、トップレベルキー `"written"` を追加する。

## 追加する構造

```json
"written": {
  "topicId1": [
    { "q": "問題文", "keywords": ["指定語句1", "指定語句2"], "a": "模範解答" },
    { "q": "問題文", "keywords": [], "a": "模範解答" }
  ],
  "topicId2": [ ... ]
}
```

- `"topics"` 配列にある**全topicIdに、それぞれちょうど2問**作る。
- 挿入位置は `"summaries"` の後（トップレベル最後）でよい。

## 作問ルール

1. **事実の根拠**: 章の元データ `C:\Users\user\projects\education-apps\marutto-study\data\content\history\{章フォルダ}\*.json` の flashcards の explanation / quiz の explanation に書かれている内容のみ。創作禁止。因果関係の記述は quiz explanation に書かれた説明が最良の材料。
2. **問題タイプ**: 定期テスト頻出の記述問題。「〜はなぜか、説明しなさい。」「〜の目的を簡単に書きなさい。」「〜のしくみを説明しなさい。」「〜と…の違いを説明しなさい。」など、理由・目的・しくみ・影響・比較を問う。単語を答えるだけの問題にしない。
3. **keywords（指定語句）**: 0〜2語。模範解答に必ずその語句を含めること。全問に付ける必要はない（半分程度に付けるのが自然）。問題文の中に「「◯◯」の語句を用いて」とは書かない（紙面側で自動表示されるため）。
4. **模範解答 `a`**: 45〜90字の1〜2文。中学生が書ける水準の簡潔な文。文末は「〜から。」（理由）や「〜こと。」「〜した。」など問いに対応した形。
5. 2問は同じトピック内で観点が重ならないようにする（例: 1問目は理由、2問目はしくみ）。

## 作業手順

1. この指示書と、担当章の `books/{章}.json` を読む。
2. 章の元データフォルダの *.json を読み、記述問題の材料（explanation の因果・目的の記述）を探す。
3. Edit で `"written"` キーを追加する（既存の volume/title/topics/timeline/summaries は一切変更しない）。
4. 編集後、`python -X utf8 -c "import json; json.load(open(r'books\{章}.json', encoding='utf-8'))"` 相当の方法でJSONが有効なことを必ず確認する。

## 最終報告

追加した問題数（topicId × 2）と、各問の観点（理由/目的/しくみ/影響/比較）の一覧を報告する。担当ファイル以外は変更しないこと。
