# つづもん 登録フロー設計（LINEファースト・中間案）

作成 2026-07-21。方針決定済み。実装は「決済は後回し・LINE受け皿を先に」(b案) で進める。

## 決定事項

- **大方針：LINEを入口にする（LINEファースト）**。ライセンス・スタ先生チャット・AI採点は
  すべて LINE uid の上に乗っているので、Web教材だけ別認証を足さず uid で一貫させる。
- **教材の閲覧ガードは「中間案」**：正規の入口を LINE リッチメニュー → LIFF に集約し、
  初回に uid でライセンス確認 → localStorage に「有効な端末」として記録 → 以降フリー閲覧。
  1コード = 3アカウント（LINE uid 単位）まで。URL 直叩きは noindex ＋ 未確認なら
  体験単元だけ表示、で緩く守る。
- **コードは覚えさせない・失わせない**（購入画面を閉じても大丈夫にする）：
  1. 主チャネルはメール（コード・手順・「LINEで受け取る」ボタン・再発行連絡先を1通に）
  2. **コードレス受け取りリンク**を正規導線にする（下記）
  3. 再発行窓口：LINEで「コードがわからない」→ 購入メール/注文番号で照合 → 再送
     （既存 `manage-tsudumon` の info/issue で対応可能）

## 登録フロー（一本道）

1. 購入（LP → Stripe）。決済完了画面＋メールで「①友だち追加 ②受け取り」を案内。
2. **受け取りリンク** `https://www.chatstudy.jp/tsudumon/activate?c=CODE`（メール・完了画面のボタン）
   を開く → LINEログイン（LIFF/LINE Login, `bot_prompt=aggressive` で友だち追加も同時）
   → uid 取得 → **コードを手入力せず自動でライセンス有効化**（既存 handleTsudumonActivation 相当）。
3. リッチメニュー「📖 教材をひらく」→ LIFF で教材（chatstudy.jp/tsudumon）。uid が取れているので
   ライセンス確認して即閲覧（ログイン画面を挟まない）。
4. 教材内で解く → 採点に納得いかなければ AI採点ボタン（同 uid、やりとりは LINE トークにも残る）。
5. 複製防止：1コード3アカウント（uid 単位）＋ DL回数制限（既存のまま）。

## 実装の進め方（b案：決済は後で差し込むだけにする）

外部依存（Stripe本番キー・価格確定：最大6SKU＋月額330円）で止まらないよう、
決済の手前を先に作る。成果物は a案（決済一気）にそのまま流用でき手戻りゼロ。

### 作るもの（順に）
1. **受け取りリンク自動有効化** `tsudumon/activate/?c=CODE` … ✅ 実装済み（2026-07-21）
   - 有効化コアを `functions/src/tsudumonActivate.ts` の `activateTsudumonLicense(uid, code)` に
     切り出し、LINE webhook の `handleTsudumonActivation` と共有（トランザクション・二重登録許容・
     上限/期限/失効検証は従来どおり）。
   - HTTP関数 `tsudumonActivate`（referenceChat と同じ idToken 検証→`line:`uid 化）を新設、
     `index.ts` で export。POST `{idToken, code}` → `{ok, planLabel, expiresLabel, already}` /
     `{ok:false, reason, message}`。
   - 受け取りページ `public/tsudumon/activate/index.html`（www.chatstudy.jp の LINE Login＝
     Firebase Auth・localStorage永続化）。`?c=CODE` を読み、未ログインは `/welcome?next=` へ、
     ログイン済みは自動で `tsudumonActivate` を叩いて結果表示（成功→教材リンク／失敗→問い合わせ）。
   - デプロイは名前指定 `functions:lineWebhook,functions:tsudumonActivate`。

2. **案内メールのひな形**（コード・手順・「LINEで受け取る」ボタン・再発行連絡先）… 未

3. **教材トップ（歴史クエスト）のライセンス確認ゲート**（中間案：初回uid確認→localStorage→3端末）… 未

### 決済（Stripe）を差し込むとき（a案の残り・キーと価格が揃ってから）
- Stripe Checkout → webhook でライセンス発行（既存の発行ロジック）＋ メール送信
- 「LINEで受け取る」= 決済セッション or 発行コードを受け取りリンクに埋める
- コードレスが失敗したときの予備として、コード手入力（LINEトーク送信）も残す
