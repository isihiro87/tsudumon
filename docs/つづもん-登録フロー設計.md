# つづもん 登録フロー設計（LINEファースト・中間案）

作成 2026-07-21。方針決定済み。実装は「決済は後回し・LINE受け皿を先に」(b案) で進める。

## 決定事項

- **大方針：LINEを入口にする（LINEファースト）**。ライセンス・スタ先生チャット・AI採点は
  すべて LINE uid の上に乗っているので、Web教材だけ別認証を足さず uid で一貫させる。
- **教材の閲覧ガードは「中間案」**：正規の入口を LINE リッチメニュー → LIFF に集約し、
  初回に uid でライセンス確認 → localStorage に「有効な端末」として記録 → 以降フリー閲覧。
  1コード = 3アカウント（LINE uid 単位）まで。URL 直叩きは noindex ＋ 未確認なら
  体験単元だけ表示、で緩く守る。
- **コードは覚えさせない・失わせない**（購入画面を閉じても大丈夫にする）：
  1. 主チャネルはメール（コード・手順・「LINEで受け取る」ボタン・再発行連絡先を1通に）
  2. **コードレス受け取りリンク**を正規導線にする（下記）
  3. 再発行窓口：LINEで「コードがわからない」→ 購入メール/注文番号で照合 → 再送
     （既存 `manage-tsudumon` の info/issue で対応可能）

## 登録フロー（一本道）

1. 購入（LP → Stripe）。決済完了画面＋メールで「①友だち追加 ②受け取り」を案内。
2. **受け取りリンク** `https://www.chatstudy.jp/tsudumon/activate?c=CODE`（メール・完了画面のボタン）
   を開く → LINEログイン（LIFF/LINE Login, `bot_prompt=aggressive` で友だち追加も同時）
   → uid 取得 → **コードを手入力せず自動でライセンス有効化**（既存 handleTsudumonActivation 相当）。
3. リッチメニュー「📖 教材をひらく」→ LIFF で教材（chatstudy.jp/tsudumon）。uid が取れているので
   ライセンス確認して即閲覧（ログイン画面を挟まない）。
4. 教材内で解く → 採点に納得いかなければ AI採点ボタン（同 uid、やりとりは LINE トークにも残る）。
5. 複製防止：1コード3アカウント（uid 単位）＋ DL回数制限（既存のまま）。

     1. build(): 各単元 view に `data-lock`（有料="2"）と `data-grade`、`__GRADE__` 定数。grade_of(ch_no)。

   - LINE Console: activate と同じ自己完結OAuthを使うので、Callback URL は各教材ドメイン（www.chatstudy.jp）配下。activate 用に登録済みだが、教材ページのパス（/tsudumon/ref/… /wb/…）を redirect_uri にするなら別途登録が要る（or redirect_uri を activate に固定して戻す設計にする＝要検討）。
