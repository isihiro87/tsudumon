# 市販ワークにおける「資料問題の画像」の使われ方 — 分析まとめ

**目的**: 我々の問題集（`pdf-workbook/`）の資料問題画像が「問題内容と噛み合っていない」という課題を解決するため、市販の中学歴史ワーク（`marutto-study/data/content/history/新ワーク1〜6`、全6分冊・約180ページ）を全ページ視覚分析し、本物の教材が画像を問題の中でどう使っているかをまとめた。

**分析対象**: 新ワーク（古代／中世／近世／近代／世界大戦／現代）＝我々と同じ「問題集」形式。必修テキスト（解説書）は役割が違うため対象外。

**注意**: 本ファイルは分析のみ。画像生成・作り直しはまだ行っていない。

## 1. 全分冊に共通する設計原則（最重要）

6分冊すべてで、以下の原則が例外なく貫かれていた。これが「画像と問題が噛み合う」ことの正体である。

### 原則① 画像は「設問が名指しする具体物」そのもの
飾りの挿絵は問題部分に一切ない。写真・地図・図はすべて「これを見て答える」ために置かれ、その画像を見なければ答えられない（または答えの手がかりになる）ように問いが作られている。

### 原則② 資料番号で設問と機械的に1対1で紐づく
「**資料1について、次の問いに答えなさい**」「資料1→(1)、資料2→(2)」という形式で、画像と設問が明示的に接続される。内容がズレる余地が構造的にない。

### 原則③ 資料の「種類」と「問い方」が固定的に対応する
| 画像の種類 | 定番の問い方 |
|---|---|
| **写真（建築・仏像・遺物・人物）** | 「この建物/像/遺物を何というか」「この人物はだれか」「何県にあるか」＝**直接同定** |
| **地図（略地図・分布図・進路図）** | 記号（A〜E／ア〜エ）や引き出し線を打ち、「この位置の国名/文明名/事件は」「◯◯の場所を記号で選べ」＝**位置特定** |
| **系図・組織図・関係図** | 枠を空欄化し「あ〜うに入る語句」＝**空欄補充**。関係図の矢印には品目・意味を問う |
| **模式図・しくみ図** | 空欄補充、または「図から分かる関係・理由を説明せよ」 |
| **グラフ** | 期間A/Bや項目ア〜エを打ち、「数値変化の理由を◯◯の語で説明」＝**読み取り＋記述** |
| **絵画・風刺画・史料（原文）** | 「描かれた場面/条文の名称は」「ここから読み取れる情勢を説明」＝**読解** |
| **表・年表** | 分類の空欄補充、期間挿入、並べ替え |

### 原則④ 同一資料を「暗記→読解→複合」と難度を上げて反復
まとめページの「資料でおぼえる」で提示した図を、そのまま練習で空欄化→定期テストで単独出題→完成問題で他資料と複合、と3〜4回使い回す。覚えた図がそのまま問われるので学習が一貫する。

### 原則⑤ 記述問題は必ず「資料を根拠に・語句を指定して」書かせる
「グラフを参考に、犠牲・賠償金を使って説明」「地形図を参考に頼朝が鎌倉を選んだ理由」など、画像が思考の起点になる。装飾ではなく解答根拠。

### 原則⑥ 役割分担が明確（一問一答には画像を置かない）
「確認問題（一問一答）」は用語の直接暗記に特化し**画像ゼロ**。画像は「練習問題」「定期テスト予想問題」＝資料読解・応用段階にのみ投入する。

### 原則⑦ 「資料マッチング」が全単元共通の花形形式
「**資料1〜3はA〜Dの文（または年表）のどれと関係が深いか選べ**」が全6分冊の定期テストに必ず登場。複数の史料・写真・風刺画を本文の出来事に結びつける。

## 3. 我々の現状（21枚）とのギャップ

現状 `books/*.json` の資料問題（歴史21枚）を、上の手本と照合した所見。

### 手本と方向性が合っているもの（型は正しい）
- `four-civilizations-map`（四大文明の地図）… 手本の古代の看板と一致。①〜④の空欄も原則③に沿う ◎
- `bakuhan-system`（幕藩体制の組織図）、`sakoku-four-windows`（四つの窓口の地図）… 近世の看板と一致 ◎
- `ritsuryo-central-org`（律令組織図）、`goon-hoko-relation`（御恩と奉公）… 中世〜奈良の看板と一致 ◎
- `triangular-trade-opium`（三角貿易）、`age-of-discovery-routes`（航路図）、`ww1-two-alliances-map`（二大陣営）、`cold-war-blocs`（冷戦陣営）、`high-growth-gnp-chart`（GDPグラフ）、`land-tax-reform`（地租改正）… いずれも各単元の定番資料と型が一致 ◎

→ **型（地図/組織図/グラフ）は概ね正しい。** 前回 gpt-image-2 で作り直したことで見栄えも手本水準になった。

### 手本とズレている／弱いもの
1. **設問との紐づけが弱い（原則②）**: 我々は画像に問題1〜2問を付けているが、手本のように「資料1→(1)、資料2→(2)」と資料番号で機械的に対応させ「資料◯について答えなさい」と明示する形になっていない。**ここが「噛み合わない」と感じる最大の原因**。画像を差し替えるより、**設問文と画像の接続を作り直す**方が効く。
2. **写真系トピックの資料が実写jpgのまま（04章）**: `horyuji.jpg`/`daibutsu.jpg` 等。手本でも写真は実写が正しい（AI生成すると実物と違うのでNG）。ここは現状維持でよいが、**新しい単元でも「建築・仏像・遺物・人物は実写、しくみ・地図・グラフはAI図解」という手本の使い分けを徹底**する。
3. **資料マッチング形式（原則⑦）が皆無**: 手本の花形「資料1〜3はA〜Dのどれと関係が深いか」が我々には一切ない。導入余地が大きい。
4. **記述問題での資料根拠（原則⑤）が弱い**: 我々の問いは名称同定中心。「グラフを参考に理由を◯◯の語で説明」型が少ない。
5. **カバーしている単元が少ない（21枚のみ）**: 手本は各単元に看板資料が必ずある。我々は未カバー単元（例: 縄文・弥生の遺物、鎌倉の金剛力士像、日清戦争の風刺画3枚、明治の就学率グラフ、戦後の農地改革帯グラフ、四大公害病の地図など）が多い。

## 4. 作り直しの方針（※画像生成はまだしない・合意用メモ）

上記ギャップから、次にやるべきことの候補。実施は別途指示を待つ。

1. **画像そのものより「設問と画像の接続」を手本化する**のが最優先（原則②）。既存21枚は型が正しいので、まず設問側を「資料◯について答えなさい」形式へ。
2. **各単元に1つ「看板資料」を持たせる**方針で、未カバー単元へ資料問題を追加（§2の★を候補リストとして使う）。
3. **画像の型を手本の対応表（原則③）に合わせて固定**する: 建築・仏像・遺物・人物＝実写、地図＝記号打ちの位置特定、組織図/系図＝空欄補充、グラフ＝期間/項目を打って読み取り、風刺画＝読解。
4. **資料マッチング問題（原則⑦）と資料根拠の記述問題（原則⑤）を新形式として導入**する。
5. AI生成が向くのは「地図・しくみ図・関係図・グラフ・（史実の）風刺画の再現」。**実在の建築・美術・人物写真はAI生成しない**（実物と違ってしまうため／手本も実写）。
